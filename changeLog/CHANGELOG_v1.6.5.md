# KaMenu v1.6.5 更新报告

## 📋 版本信息
- **版本号**: 1.6.5
- **发布日期**: 2026年7月20日

---

## ✨ 新增功能

### 1. 新增 ESC 暂停菜单入口

KaMenu 现在可以通过原版数据包，在客户端 ESC 暂停菜单中注册一个自定义入口按钮：

```bash
/km pause register
/km pause unregister
/km pause info
```

- `register`：读取 `plugins/KaMenu/pause_menu.yml` 并生成入口数据包
- `unregister`：移除 KaMenu 生成的数据包
- `info`：显示源文件和数据包路径

数据包写入主世界根目录：

```text
world/datapacks/KaMenuPauseEntry
```

执行 `register` 或 `unregister` 后需要完整重启服务器。KaMenu 固定注册一个 ESC 入口；需要多个功能时，可在入口 Dialog 内使用按钮矩阵。

### 2. 新增 `pause_menu.yml` 静态 Dialog

插件启动时会检查 `plugins/KaMenu/pause_menu.yml`。文件不存在时自动释放默认模板，已经存在的文件不会被覆盖。

支持以下结构：

- `Title` 与 `External-Title`
- `Settings.can_escape`、`Settings.pause`、`Settings.after_action`
- `Body.message` 静态消息
- `Body.item` 静态物品展示
- `Bottom.notice` 单按钮
- `Bottom.confirmation` 确认/取消双按钮
- `Bottom.multi` 多按钮矩阵和 `exit`

示例：

```yaml
Title: '&b服务器菜单'
External-Title: '服务器菜单'

Settings:
  can_escape: false
  pause: false
  after_action: NONE

Body:
  welcome:
    type: message
    width: 300
    text: '&f请选择下方功能。'

Bottom:
  type: multi
  columns: 2
  buttons:
    main:
      text: '&a打开主菜单'
      menu: 'example/main_menu'
    website:
      text: '&b官方网站'
      url: 'https://example.com'
  exit:
    text: '&c关闭'
```

### 3. 暂停菜单按钮支持多种操作

每个按钮可配置以下一种目标：

- `menu`：回调 KaMenu 并打开指定菜单
- `actions`：回调 KaMenu 并执行标准动作列表
- `url`：由客户端打开 URL
- `copy`：由客户端复制文本
- `command`：由客户端以玩家身份执行命令

`menu` 与 `actions` 使用固定按钮 ID 回调服务端。客户端不会收到真实菜单 ID 或动作列表，实际操作以服务端注册时缓存的 `pause_menu.yml` 为准。

### 4. 支持 Inputs 和服务端动作上下文

暂停菜单支持四种标准输入组件：

- `input`：文本输入框
- `slider`：数值滑块
- `dropdown`：单项选择按钮
- `checkbox`：复选框

按钮点击后，KaMenu 会取得实际点击玩家及客户端 `DialogResponseView`。配置了 `actions` 的按钮可通过 `$(key)` 读取输入，并继续使用 PAPI、KaMenu 内部变量、条件、JavaScript 和动作包。

```yaml
Inputs:
  player_name:
    type: input
    text: '&e玩家名'
    max_length: 16
    remove_chars: global

  amount:
    type: slider
    text: '&e数量'
    min: 1
    max: 10
    step: 1
    default: 1

Bottom:
  type: notice
  confirm:
    text: '&a提交'
    actions:
      - 'tell: &a玩家：%player_name%，输入：$(player_name)，数量：$(amount)'
      - 'close'
```

按钮 `actions` 使用完整 KaMenu 动作执行器，支持：

- 嵌套条件及 `allow` / `deny`
- `wait` 与 `return`
- JavaScript
- 全局 actions 包
- 同一 `pause_menu.yml` 中的 `Events.Click` 动作包
- `open`、`close`、指令、消息、音效等现有动作

`reset` 不适用于静态暂停菜单，因为该配置不属于 `MenuManager`；需要进入动态菜单时请使用 `open:`。

## 🔒 安全与可靠性

- 数据包生成前会校验 YAML、按钮目标和生成的 JSON
- 输入组件会复用普通菜单的首尾空格与 `remove_chars` 清理逻辑
- 禁止同一按钮同时配置多个目标，避免操作歧义
- 自动定位包含 `level.dat` 的主世界根目录，避免把数据包写入维度子目录
- 自动复用服务器 Bukkit 数据包的格式范围，兼容新版数据包元数据要求
- 启动时会从 `pause_menu.yml` 恢复可信按钮目标，不信任客户端提交的动作内容

## 📝 兼容说明

- `pause_menu.yml` 的标题、Body、输入标签和按钮文本为静态内容，不解析运行期变量
- 运行期变量、PAPI、条件和 JS 应写在按钮 `actions` 内
- `url`、`copy` 和 `command` 是客户端静态动作，不会把 Inputs 提交给 KaMenu 动作系统
- 配置文件版本保持 `5`，无需升级现有 `config.yml`

5. 再次完整重启服务器，使 ESC 入口数据包生效

# KaMenu v1.6.5 Update Report

## 📋 Version Info
- **Version**: 1.6.5
- **Release Date**: July 20, 2026

## ✨ New Features

### 1. ESC pause screen entry

KaMenu can now register a custom entry button on the vanilla ESC pause screen through a datapack:

```bash
/km pause register
/km pause unregister
/km pause info
```

- `register`: reads `plugins/KaMenu/pause_menu.yml` and generates the entry datapack
- `unregister`: removes the datapack generated by KaMenu
- `info`: displays the source file and datapack paths

The datapack is written to the main world root:

```text
world/datapacks/KaMenuPauseEntry
```

A full server restart is required after `register` or `unregister`. KaMenu registers one ESC entry; use a button matrix inside its Dialog when multiple operations are needed.

### 2. Static `pause_menu.yml` Dialog

KaMenu checks `plugins/KaMenu/pause_menu.yml` during startup. The default template is released when the file is missing, while existing files are never overwritten.

Supported structures include:

- `Title` and `External-Title`
- `Settings.can_escape`, `Settings.pause`, and `Settings.after_action`
- Static `Body.message`
- Static `Body.item`
- Single-button `Bottom.notice`
- Two-button `Bottom.confirmation`
- `Bottom.multi` button matrices and `exit`

Example:

```yaml
Title: '&bServer Menu'
External-Title: 'Server Menu'

Settings:
  can_escape: false
  pause: false
  after_action: NONE

Body:
  welcome:
    type: message
    width: 300
    text: '&fSelect an option below.'

Bottom:
  type: multi
  columns: 2
  buttons:
    main:
      text: '&aOpen Main Menu'
      menu: 'example/main_menu'
    website:
      text: '&bWebsite'
      url: 'https://example.com'
  exit:
    text: '&cClose'
```

### 3. Multiple pause-menu button operations

Each button may define one target:

- `menu`: calls KaMenu and opens a menu
- `actions`: calls KaMenu and executes a standard action list
- `url`: opens a URL on the client
- `copy`: copies text on the client
- `command`: runs a command as the player on the client

`menu` and `actions` use fixed button IDs for server callbacks. Real menu IDs and action lists are never sent to the client; operations are resolved from the server-side `pause_menu.yml` state loaded during registration.

### 4. Inputs and server-side player context

Pause menus support four standard input types:

- `input`: text input
- `slider`: numeric slider
- `dropdown`: single-option button
- `checkbox`: checkbox

After a click, KaMenu obtains the actual player and the client's `DialogResponseView`. Buttons with `actions` can read values through `$(key)` and use PAPI, KaMenu variables, conditions, JavaScript, and action packages.

```yaml
Inputs:
  player_name:
    type: input
    text: '&ePlayer Name'
    max_length: 16
    remove_chars: global

  amount:
    type: slider
    text: '&eAmount'
    min: 1
    max: 10
    step: 1
    default: 1

Bottom:
  type: notice
  confirm:
    text: '&aSubmit'
    actions:
      - 'tell: &aPlayer: %player_name%, input: $(player_name), amount: $(amount)'
      - 'close'
```

Button `actions` use the complete KaMenu action engine, including:

- Nested conditions with `allow` / `deny`
- `wait` and `return`
- JavaScript
- Global action packages
- Local `Events.Click` packages in the same `pause_menu.yml`
- Existing actions such as `open`, `close`, commands, messages, and sounds

`reset` does not apply to a static pause menu because it is not owned by `MenuManager`; use `open:` to enter a dynamic menu.

## 🔒 Security and Reliability

- YAML, button targets, and generated JSON are validated before datapack output
- Inputs reuse regular-menu edge-space trimming and `remove_chars` cleanup
- A button cannot define multiple targets
- The datapack path is resolved from the main world root containing `level.dat`
- Bukkit's datapack format range is reused for newer metadata requirements
- Trusted button targets are restored from `pause_menu.yml` during startup; client-submitted action content is never trusted

## 📝 Compatibility Notes

- Existing menus, `/km open`, MenuManager, and regular Dialog behavior are unchanged
- Titles, Body content, input labels, and button labels in `pause_menu.yml` are static and do not resolve runtime variables
- Runtime variables, PAPI, conditions, and JavaScript belong in button `actions`
- Client-side `url`, `copy`, and `command` actions do not submit Inputs to KaMenu's action engine
- The config version remains `5`; existing `config.yml` files require no migration

## 🚀 Upgrade Guide

### Upgrading from 1.6.4 to 1.6.5

1. Replace the plugin jar with `KaMenu-1.6.5.jar`
2. Fully restart the server so KaMenu can release the default `pause_menu.yml`
3. Edit `plugins/KaMenu/pause_menu.yml`
4. Run `/km pause register`
5. Fully restart the server again to load the ESC entry datapack
